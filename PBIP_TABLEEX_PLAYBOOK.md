# PBIP TableEx Playbook

## Purpose

Document the validated workflow for adding semantic-model measures to PBIP `tableEx` visuals without breaking Desktop load.

Validated on:

- report: `analisis_disponibilidad.pbip`
- page: `02_Top_Venta_12M`
- visual: `v_tabla_top_venta_12m`

## Mandatory sequence

1. Create or update the measure.
2. Verify it exists in the live Desktop model.
3. Verify it exists in TMDL on disk.
4. Test it with a small DAX query.
5. Add one column to the `tableEx` visual.
6. Validate `report.json`.
7. Reopen Desktop.
8. Document the result.

## Persistence rule

Do not assume a measure is persisted just because it works in the live session.

Before editing `report.json`, verify:

- live model: OK
- TMDL on disk: OK

## Safe `tableEx` serialization pattern

- `projections.Values[]`:
  - `MEDIDAS_MASTER_DISPONIBILIDAD.<Measure Name>`
- `prototypeQuery.From[]`:
  - base table
  - measures table
- `prototypeQuery.Select[]`:
  - `Measure` node
  - `SourceRef` must point to the measures table alias

Copy the structure generated by Desktop. Do not invent it.

## Safe Top 10 filter pattern

Validated filter:

- field: `Ranking Venta 12M Global`
- comparison: `< 11`

Serialized as:

- `visualContainers[].filters`
- `type = Advanced`
- `ComparisonKind = 3`
- `Right.Literal.Value = 11L`
